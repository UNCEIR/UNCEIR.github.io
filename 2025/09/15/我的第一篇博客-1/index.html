JUC并发编程01：进程和线程

2025/9/15

进程和线程

1.概念的讲述

首先分清这两个之间的区别非常重要。

	进程是操作系统中的应用程序、是资源分配的基本单位。
	线程是用来执行具体的任务和功能，是CPU调度和分派的最小单位。
	一个进程往往可以包含多个线程，至少包含一个。

1.1进程

一个进程往往可以包含多个线程，至少包含一个。
Java默认有两个线程：一个 main， 一个GC。

1.2线程

对于Java而言：Thread、Runable、Callable 是进行开启线程的。java是没有权限去开启线程的、操作硬件的，这是一个 native 的一个本地方法，它调用的底层是c++代码。只不过线程的创建与管理权限是操作系统（OS）内核提供的。Java 本身不能“直接”去操作 CPU 或硬件，而是通过 JVM（Java 虚拟机） 和 操作系统的线程 API 来间接完成的。

1.3并发

  • 多线程操作同一个资源
  • CPU只有一核，模拟出来多条线程。可以使用CPU快速交替，来模拟多线程。
  • 并发编程的本质：充分利用CPU的资源。
  • 类比：一个拉面师傅同时接了 10 个订单，他不会一下子做完一个再做下一个，而是 揉一会面、煮一会汤、切一会菜，在多个任务之间来回切换，看起来像是“同时”在干很多事。

1.4并行

  • CPU多核，多个线程可以同时执行。 我们可以使用线程池
  • 类比：有 3 个厨师，每人同时炒一个菜，3 个菜可以真正同时出锅。

1.5线程的状态

public enum State {
    // 线程新生
    NEW,

  	//运行状态
    RUNNABLE,

    //阻塞
    BLOCKED,

    //等待，死死地等
    WAITING,

    //超时等待
    TIMED_WAITING,

    //终止
    TERMINATED;
}

1.6 wait和sleep的区别

来自不同的类：

  • wait -> Object
  • sleep -> Thread

锁的释放

  • wait 会释放锁。
  • sleep 处于睡眠状态，不会释放锁。

使用的范围不同

  • wait 必须在同步代码块中。
  • sleep 可以在任何地方处于睡眠。

是否需要捕获异常

  • wait 不需要捕获异常。
  • sleep 必须要捕获异常。
